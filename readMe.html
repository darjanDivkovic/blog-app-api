## Setting up a Django project 1. Create a virtual environment using the following command: python3 -m venv django_rest_api 2. Once the process is done, cd to the created django_rest_api directory: cd django_rest_api 3. Now, activate the virtual environment. If you are on Windows, use command django_rest_api/Scripts/activate, or source django_rest_api/bin/activate 4. Install Django to your environment: python -m pip install django 5. Initialize and start the Django REST project: django-admin startproject backend |-----------------------------------------------------------| | django_rest_api_tasks, a top level project folder | | django_rest_api_tasks/backend, | | a lower level project folder for managing apps | | manage.py, the command center file for your project | |-----------------------------------------------------------| ** INSTALL WORKED ** Apps are a lower level units that represent a specific module. For example, in our project, we will create an app called tasks. To do so, change the directory to the backend 6. Create the app [post - for blog posts] python manage.py startapp posts 7. Add APP tasks to the INSTALLED_APPS list in the backend/settings.py file: INSTALLED_APPS = [ + 'tasks' ] ** We can now start setting up a Django REST API. ** ## Creating a task model
